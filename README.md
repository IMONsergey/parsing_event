# Сбор B2B-контактов для BAEV

Этот репозиторий — GitHub-based pipeline для сбора и нормализации целевых B2B-контактов потенциальных клиентов и партнёров BAEV.

BAEV занимается презентациями, коммуникационным дизайном, упаковкой смыслов, структурой выступлений, коммерческих предложений, питчей, корпоративных презентаций, публичных материалов и визуальной коммуникации для бизнеса.

GitHub используется как source of truth. Google Sheets используется как витрина для просмотра, фильтрации и ручной работы с итоговой таблицей.

## Целевые сегменты

Проект ориентирован на сегменты, которым регулярно нужны презентации, питчи, коммерческие материалы, публичные выступления и визуальная упаковка смыслов:

- event-агентства;
- presentation design studios и pitch deck agencies;
- коммуникационные агентства;
- PR-агентства;
- маркетинговые агентства;
- брендинговые агентства;
- digital-агентства;
- продюсерские агентства;
- организаторы форумов, конференций и бизнес-мероприятий;
- corporate communications команды;
- HR / employer brand команды;
- business development / sales enablement команды;
- консалтинговые компании;
- образовательные и бизнес-сообщества;
- компании, которым регулярно нужны презентации для продаж, инвесторов, выступлений и внутренних коммуникаций.

Допустимые значения поля `Сегмент`:

`event`, `presentations`, `communications`, `pr`, `marketing`, `branding`, `digital`, `production`, `corporate`, `consulting`, `education`, `sales`, `hr`, `community`, `other`.

## Структура проекта

- `config/sources.csv` — публичные источники для автоматического сбора.
- `config/manual_contacts.csv` — контакты, добавленные вручную.
- `config/exclude_domains.csv` — домены личной почты, которые нужно помечать для проверки.
- `config/keywords.csv` — ключевые слова релевантности по сегментам.
- `data/contacts_ru.csv` — итоговая таблица на русском языке.
- `data/contacts_ru.json` — JSON-версия итоговой таблицы.
- `data/run_report.json` — отчёт последнего запуска.
- `data/email_quality_report.json` — отчёт качества email.
- `data/company_quality_report.json` — отчёт по уникальным компаниям и нескольким контактам.
- `data/source_quality_report.json` — отчёт качества источников.
- `src/leads_pipeline.py` — основной скрипт.
- `.github/workflows/collect-leads.yml` — ежедневный запуск через GitHub Actions.

## Как добавить источник

Откройте `config/sources.csv` и добавьте строку с колонками:

```csv
Активен,Сегмент,Тип источника,Источник,Компания,Сайт,URL источника,Страна,Город,Отрасль,Приоритет,Комментарий
```

Скрипт обрабатывает только строки, где `Активен` равно `да`.

В `URL источника` указывайте только публично доступную страницу: страницу контактов, страницу агентства, профиль в каталоге, страницу мероприятия или страницу компании.

## Как добавить контакт вручную

Откройте `config/manual_contacts.csv` и добавьте строку с колонками:

```csv
Сегмент,Компания,Имя,Должность,Email,Телефон,Сайт,LinkedIn,Telegram,Страна,Город,Отрасль,Источник,URL источника,Приоритет,Комментарий
```

Ручные контакты объединяются с автоматически найденными. При дублях главный ключ — `Email`; если email пустой, используется сочетание `Компания + Сайт + URL источника`.

## Как запустить локально

```bash
pip install -r requirements.txt
python src/leads_pipeline.py
```

После запуска обновятся:

- `data/contacts_ru.csv`;
- `data/contacts_ru.json`;
- `data/run_report.json`;
- `data/email_quality_report.json`;
- `data/company_quality_report.json`;
- `data/source_quality_report.json`.

Можно указать нестандартные пути:

```bash
python src/leads_pipeline.py \
  --sources config/sources.csv \
  --manual config/manual_contacts.csv \
  --output data/contacts_ru.csv \
  --json-output data/contacts_ru.json \
  --report data/run_report.json
```

## Как запустить через GitHub Actions

Workflow `Собрать контакты BAEV` находится в `.github/workflows/collect-leads.yml`.

Он запускается:

- вручную через `workflow_dispatch`;
- автоматически один раз в сутки в 06:00 UTC.

Workflow:

1. забирает репозиторий;
2. устанавливает Python 3.11;
3. устанавливает зависимости из `requirements.txt`;
4. запускает `python src/leads_pipeline.py`;
5. коммитит изменения в `data/contacts_ru.csv`, `data/contacts_ru.json`, `data/run_report.json`, `data/email_quality_report.json`, `data/company_quality_report.json`, `data/source_quality_report.json`, если они появились.

Если изменений нет, workflow выводит `No changes`.

## Как подключить Google Sheets

На первом этапе Google Sheets подключается как витрина через CSV из GitHub.

В ячейку `A1` можно вставить:

```text
=IMPORTDATA("https://raw.githubusercontent.com/IMONsergey/parsing_event/main/data/contacts_ru.csv")
```

Если Google Sheets некорректно определит кодировку или разделитель, используйте ручной импорт:

`Файл → Импорт → По URL / Загрузить CSV`

Raw CSV:

https://raw.githubusercontent.com/IMONsergey/parsing_event/main/data/contacts_ru.csv

## Как потом подключить прямую запись в Google Sheets

Прямая запись сейчас не реализована. Для отдельного этапа потребуется:

- Google Service Account;
- доступ service account к нужной таблице;
- GitHub Secrets:
  - `GOOGLE_SERVICE_ACCOUNT_JSON`;
  - `GOOGLE_SPREADSHEET_ID`;
  - `GOOGLE_WORKSHEET_NAME`;
- отдельный скрипт `sync_to_google_sheets.py`.

До этого момента GitHub остаётся source of truth, а Google Sheets читает опубликованный CSV.

## Ограничения

- Проект не отправляет письма автоматически.
- Проект не делает автоматическую email-рассылку.
- Проект не обходит авторизацию, капчи, paywall, закрытые зоны и технические ограничения сайтов.
- Проект собирает данные только с публично доступных страниц и вручную заданных источников.
- Проект не использует Playwright или Selenium на первом этапе.
- Проект не хранит секреты в репозитории.
- Проект не пишет напрямую в Google Sheets без отдельного этапа и credentials.
- Личные email не удаляются автоматически: они помечаются как `Личный`, а статус становится `Проверить`.

## Поля итоговой таблицы

`data/contacts_ru.csv` всегда содержит русские заголовки:

```text
ID,ID компании,Ключ компании,Повтор компании,Кол-во контактов компании,№ контакта в компании,Статус,Приоритет,Сегмент,Тип компании,Компания,Имя,Должность,Email,Тип email,Телефон,Сайт,LinkedIn,Telegram,Страна,Город,Отрасль,Источник,URL источника,Потенциальный интерес,Дата добавления,Дата обновления,Уверенность,Комментарий
```

Каждый контакт сохраняет источник и конкретный URL, откуда взяты данные. Это важно для дальнейшей ручной проверки и ручной рассылки.

Одна компания может встречаться в итоговой таблице несколько раз, если у неё найдено несколько разных валидных email. Это сделано специально, чтобы не терять разные точки входа: общий адрес, адрес продаж, партнёрский адрес или корпоративный контакт сохраняются отдельными строками.

Такие строки помечаются колонками `ID компании`, `Ключ компании`, `Повтор компании`, `Кол-во контактов компании` и `№ контакта в компании`. Одинаковые email строго удаляются, но разные email одной компании сохраняются. Google Sheets через `IMPORTDATA` подтянет новые колонки автоматически из raw CSV.
